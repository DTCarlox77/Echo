Markdown Regular

Esta herramienta convierte expresiones Markdown a su equivalente en HTML, permitiendo establecer propiedades como tamaño, color, fondo y fuente en etiquetas de texto.


Etiquetas

Puedes utilizar todas las etiquetas de HTML para personalizar tus textos.

Sintaxis de las etiquetas

Las etiquetas se delimitan con el uso de paréntesis. Antes de nombrar una etiqueta, es necesario utilizar el comando /rmd.

Para llamar a una etiqueta, primero abrir y cerrar paréntesis '()', luego colocar un par de corchetes al final '[]', entre estos estarán ubicadas las propiedades. Aunque no se establezcan propiedades, siempre es necesario colocar dichos corchetes. Dentro de los paréntesis se pueden agregar la cantidad de etiquetas deseadas, sin embargo, estas etiquetas heredarán siempre el estilo asignado en las propiedades.

(etiqueta1(contenido) etiqueta2(contenido))[]

Ejemplos de uso

Si se desea mostrar un texto similar al siguiente dentro del chat:

Hello Echo

Puede hacerlo de la siguiente manera:

/rmd (h1(Hello Echo))[]

Si se desea mostrar una lista similar al siguiente dentro del chat:

  • Elemento 1
  • Elemento 2
  • Elemento 3

Puede hacerlo de la siguiente manera:

/rmd (ul(li(Elemento 1) li(Elemento 2) li(Elemento 3)))[]

Puedes investigar más información acerca de las etiquetas HTML en la documentación oficial.


Propiedades

Actualmente, esta sintaxis permite el uso de cuatro propiedades para dar estilo a tus mensajes. Estas son:

  • col: Recibe un color directamente por su nombre en inglés (red, green, blue...), rgb, hexadecimal y rgba no están disponibles actualmente.
  • bgc: Al igual que la propiedad anterior, establece un color, solo que este será para el fondo del texto (resaltado en otras palabras).
  • siz: Siz establecerá un tamaño en unidades de píxeles a tu texto. Deberás ingresar un número entero.
  • fon: Establecerá la tipografía en la que quieras que se muestre tu texto. La correcta renderización dependerá de tu navegador.

Cada etiqueta podrá utilizar como máximo una propiedad diferente. En otras palabras, no puedes agregar dos veces la propiedad col en un texto.

Sintaxis de las propiedades

Las propiedades siempre estarán ubicadas en el interior de dos corchetes, a la par del paréntesis (representantes de etiquetas) con el que se cierra una etiqueta:

(etiqueta(contenido))[col='color de preferencia', bgc='color de preferencia', siz='tamaño de preferencia', fon='fuente de preferencia']

Ejemplos de uso

Si se desea mostrar un texto similar al siguiente dentro del chat:

Hello Echo

Puede hacerlo de la siguiente manera:

/rmd (h1(Hello Echo))[col=indigo, siz=30]

Si se desea mostrar un texto similar al siguiente dentro del chat:

Esto es un texto resaltado

Puede hacerlo de la siguiente manera:

/rmd (p(Esto es un texto resaltado))[col=lightgreen, fon=sans-serif]

¿Por qué Echo?

El nombre 'Echo' ha sido elegido para reflejar la esencia de la aplicación. Al igual que un eco que resuena y se propaga, Echo representa la idea de que cada interacción, cada conexión y cada contribución en la plataforma resuena y se refleja en la red social.


Comandos de la aplicación

Echo permite la utilización de comandos en las salas de chat, para que se ejecuten de forma correcta estos deberán ir posisionados al inicio de tu mensaje.

  • /eb - Utiliza la inteligencia artificial integrada. Esta toma la información de la API de OpenAI.
  • /md - Envía mensajes en formato de Markdown.
  • /rmd - Envía mensajes con estilos CSS en etiquetas HTML dentro de los contenedores Markdown.

Mensajes en la sala

Cada sala tiene una capacidad máxima de 30 mensajes. Una vez que se alcanza este límite, cualquier mensaje adicional será eliminado de manera permanente para mantener la cantidad de mensajes en la sala dentro del límite establecido.
